Father program is the entry program for Ex1. To accomplish the task described in EX1 instruction, Father program will execute the following steps:

  1. Parse the command line arguments – makes sure there is a VALID input. [Extra: Add -h, –help support flags to explain the usage]
  2. Open the input file given as an argument, the file is expected to be have a valid format. If opening the file fails, the program should exit with error status described under common_types and print the relevant error message. Read the following parameters from the file: forest_size, time_steps (= generations), forest_string.
    1. Create output.txt file (if already in the folder, overwrite). Handle any errors that can occur.

Loop over time_steps: 

  4. Call Son program as a child process - it should count the 
  number of fires in the forest.

  5. Print forest string to output.txt.
  Print number of fires to output.txt.

  6. Advance in time-step: meaning changing the forest status 
  according to instructions logic.
  1. Program cleanup: close and free all resources - files/allocated memory. Return 0 on success, otherwise ERROR code.
