errors_e update_forest_status ( char *  p_forest_string_char,
const uint32_t  u_forest_size 
)

update_forest_status takes the current forest state and updates it –inplace– according to the following logic:

  • All trees near Fire, are turning to Fire
  • All ground cells with atleast two neighboring cells ( INCLUDING DIAGNALS) are turning to Tree
  • All fires becomes Ground

example: size = 3, T G F T G T G F T

==> T T G T T F G G F

Parameters
[in/out]p_forest_string_char: string representing the forest - hold 'F','G','T' charecters and ends with '\0'
[in]u_forest_sizesize of the forest.
Returns
errors_e status of the function, 0 for success and ERROR codes otherwise
